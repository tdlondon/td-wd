what is a form

  • interface to collect data
  • allows users to provide data
  • data is then sent to the server
  • forms are everywhere, you use them to login,
    register, buy products and much more
  • forms contain elements paired with labels and grouped by fieldsets

form tag

  • contains form elements
  • has a name
  • action specifies the script that will handle data
  • method sets how data will be sent (get / post)

<form name="myForm" action="path/to/server-side/script.php" method="get">
    ...
</form>

fieldsets

  • are used to group form elements
  • appear outlined in light grey
  • can be described by legend tags

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
        <legend>fieldset description</legend>
    </fieldset>

    <fieldset>
        <legend>fieldset description</legend>
    </fieldset>
</form>

input tags

  • paired and described by labels
  • inline elements
  • do not need closing tags
  • identified by a name
  • collect a value
  • can be described by a placeholder (html5)
  • defined by a type: text, password, number, date, radio etc..

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
        <legend>fieldset description</legend>
        <div>
            <label>role of the input field</label>
            <input type="text" name="field" placeholder="format" value=""/>
        </div>
    </fieldset>
</form>

radio buttons

  • input tags with radio type
  • can be turned on or off
  • belong to a group
  • selecting one will turn other radio buttons of the same group off
  • have a name, a value
  • connected radio buttons need the same name
  • we need text outside checkboxes to describe them
  • can be disabled or checked

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
    <legend>radio buttons</legend>
    <div>
        <label>select A or C</label><br>
        <input type="radio" checked name="letter" value="A"/> A <br>
        <input type="radio" disabled name="letter" value="B"/> B <br>
        <input type="radio" name="letter" value="C"/> C
    </div>
    </fieldset>
</form>

checkboxes

  • input tags with checkbox type
  • can be turned on or off
  • belong to a group
  • users can select more than one options
  • have a name, a value
  • connected checkboxes need the same name
  • we need text outside radio buttons to describe them
  • can be disabled or checked

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
        <legend>checkboxes</legend>
        <div>
            <label>select one or more letters out of a list</label><br>
            <input type="checkbox" checked name="letter" value="A"/> A <br>
            <input type="checkbox" disabled name="letter" value="B"/> B <br>
            <input type="checkbox" name="letter" value="C"/> C <br>
        </div>
    </fieldset>
</form>

drop down menus

  • collapsible lists of options
  • option tags within select
  • can be disabled or selected
  • select tags have names, options have values
  • can use multiple attribute to allow multiple selection

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
    <legend>drop down menus</legend>
        <div>
        <label>select letters from a drop down menu</label><br>
        <select name="letters" multiple>
            <option selected value="A">A</option>
            <option disabled value="B">B</option>
            <option value="C">C</option>
            <option value="D">D</option>
            <option value="E">E</option>
        </select>
        </div>
    </fieldset>
</form>

textareas

  • element to collect longer texts
  • have a name but no value
  • can have a placeholder
  • values are collected in between textarea tags
  • used to collect messages, email texts, comments etc..

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
    <legend>textarea</legend>
    <div>
        <label>write here your comment</label><br>
        <textarea name="comment"
            placeholder="your comment here..."></textarea>
    </div>
    </fieldset>
</form>

buttons

  • clickable inline level elements to trigger a response or submit forms
  • can be input tags with button type
  • can be button tags with submit, reset, button type
  • can be disabled

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
    <legend>buttons</legend>
    <div>
        <button type="reset">clears form</button><br>
        <button type="submit">submits form</button><br>
        <button type="button">general button</button><br>
        <input type="button" value="general button" disabled/>
    </div>
    </fieldset>
</form>

form methods and submitting forms

  • most common form methods are get and post
  • data is collected as key-value pairs
  • get generates a query string after page url, post does not
  • query strings will have ?name=value&name2=value2 structure
  • query strings will expose form data in url bar, do not use get for sensitive information
  • form data is sent to the value of action attribute

<form name="myForm" action="path/to/server-side/script.php" method="get">
    <fieldset>
    <legend>submitting forms</legend>
    <div>
        <input type="text" name="key" value="what you typed in this field"/>
        <button type="submit">submits form</button><br>
    </div>
    </fieldset>
</form>
